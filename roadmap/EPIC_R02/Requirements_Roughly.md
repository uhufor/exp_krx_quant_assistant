# Quant Assistant GUI

- 상태: 초안, 미승인

## 용어
- v1: 일일 퀀트 어시스턴트 (지정된 관심종목을 정해진 퀀트 전략으로 실행 후 보고서를 텔레그램으로 전송하는 기능)
- v1.5: 노코드 전략 워크스페이스 (코드 수정 없이 CLI 및 JSON을 이용하여 팩터, 공식, 규칙, 전략을 생성 및 관라하고 백테스트를 실행해볼 수 있는 기능)

## 목표
- v1.5의 구현으로 노코드 전략 워크스페이스 기반으로 기능을 GUI 화면을 제공하여 UI상에서 새로운 전략을 생성 및 테스트 해볼 수 있는 기능 구현

## 상세 요구사항
- GUI는 CLI에서 제공하는 모든 기능을 사용 가능해야 함
- JSON 파일을 입력을 하지 않고 제공된 UI를 가지고 팩터, 공식, 규칙, 전략 생성이 가능해야 함. (즉 UI나 중간 도메인단에서 적절한 JSON을 생성하여 전달하는 구조가 필요할 수 있음)
- 신규 팩터를 생성, 수정, 삭제 등 관리 기능이 있어야함
- 신규 공식을 생성, 수정, 삭제 등 관리 기능이 있어야함.
- 신규 규칙을 생성, 수정, 삭제 등 관리 기능이 있어야함.
- 신규 전략을 생성, 수정, 삭제 등 관리 기능이 있어야함.
- 전략을 기반으로 특정 종목에 대해서 백테스트 기능이 있어야함.
- 백테스트는 종목명 또는 종목코드, 시작일, 종료일, 데이터 소스(krx, fixture), 적용 전략을 입력 받아 실행 가능해야 함.
- UI를 제공하기 가장 적절한 프레임워크 및 아키텍쳐는 적절한 방법으로 결정 (예시: 웹서버+API 서버로 구성하여 웹베이스로 구현)
- 기존 CLI 명령어도 계속 지원해야 함.

## 설계 및 코딩 규칙
- 응집도는 높고 결합도는 낮은 설계 적용
- 코드 간결함, 가독성을 높이는 코드 지향
- 언어 표준 코딩 컨벤션 준수

## 테스트
- CLI 기능은 기존 테스트를 모두 통과
- GUI 기능 관련해서 필요시 테스트 추가
